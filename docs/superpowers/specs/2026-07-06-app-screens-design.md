# 앱 화면/내비게이션 구조 설계

## 배경

캘린더에 등록된 여행 일정을 기준으로 날씨·재난 정보를 알려주는 서비스. 최초 구현 범위로는 하단 탭
내비게이션과 전체 화면 구조를 먼저 잡고, 실제 날씨/재난 데이터 연동은 화면별로 이후 별도 설계에서
진행한다.

## 기능 구현 순서(로드맵)

1. 캘린더 연동 & 여행 일정 추출
2. 날씨 알림 MVP (기상청 단기/중기예보)
3. 기상특보 알림 (호우/폭염/강풍/풍랑/태풍)
4. 위치 기반 재난 위험 (하천 수위 - WAMIS, 해안 - 국립해양조사원, 공간정보 - V-World)

이 문서는 위 로드맵과 별개로, 전체 화면 구조와 하단 탭 내비게이션 구현 방식을 다룬다.

## 내비게이션 구조

- 라이브러리: React Navigation (`@react-navigation/native`, `@react-navigation/bottom-tabs`,
  `react-native-screens`)
- 하단 탭은 **지도**, **캘린더** 2개로 고정한다. 홈 요약, 설정, 알림 히스토리 등은 탭을 늘리지 않고
  두 탭 안의 하위 화면이나 헤더 아이콘으로 접근한다. (탭이 늘어나면 각 탭의 역할이 흐려진다고 판단)
- 각 탭은 필요 시 내부적으로 스택 내비게이션을 가져 상세 화면으로 이동할 수 있다.

## 화면 인벤토리

### 온보딩 (최초 실행 1회)
1. 캘린더 접근 권한 요청 화면
2. 연동할 캘린더 선택 화면 (계정/캘린더가 여러 개인 경우)
3. 완료 후 캘린더 탭으로 진입

### 캘린더 탭
- **일정 목록 화면**: 연동된 일정 중 여행으로 판단되는 항목을 카드로 나열. 카드에는 제목/기간/장소와
  위험도 배지(예: 폭염주의보)를 표시. 우측 상단에 설정(⚙) 아이콘.
- **일정 상세 화면** (카드 탭 시 이동): 해당 여행 기간의 날짜별 날씨 예보, 발효 중인 특보 목록,
  "지도에서 보기" 버튼 → 지도 탭으로 해당 위치 정보를 가지고 이동.

### 지도 탭
- **지도 화면**: Kakao Map 위에 V-World 하천 레이어를 표시. 상단에 어떤 여행 일정을 볼지 선택하는
  드롭다운(일정이 여러 개일 때 전환용). 하천/해안 근처 위험 지점은 마커로 표시.
- **위험 상세 팝업** (마커 탭 시 바텀시트): 하천명, 현재 수위, 위험 임계값 대비 상태.

### 설정 화면
- 진입 경로: 캘린더 탭 우측 상단 아이콘 (별도 탭 아님)
- 연동 캘린더 변경/해제
- 알림 on/off, 알림 리드타임(예: 여행 D-1 알림) 설정

### 알림(푸시) 처리
- 별도 알림 히스토리 화면은 MVP 범위에서 제외한다.
- 푸시 알림을 탭하면 해당 일정의 **일정 상세 화면**으로 딥링크 이동한다.

## 이번 구현 범위 (탭 내비게이션 스캐폴딩)

- `App.tsx`에 `NavigationContainer` 적용
- `src/navigation/AppNavigator.tsx`에 `createBottomTabNavigator()`로 `Map`, `Calendar` 탭 등록
- `src/screens/MapScreen.tsx`, `src/screens/CalendarScreen.tsx`는 위 화면 인벤토리를 채워 넣기 전
  placeholder 상태로 유지
- 온보딩, 일정 상세, 설정, 위험 상세 팝업 등 나머지 화면은 이 스캐폴딩 위에 이후 각각 별도로 설계/구현

## 테스트

- `AppNavigator`가 정상 렌더링되는지 확인하는 스모크 테스트를 `__tests__`에 추가

## 범위 제외

- 캘린더 연동 로직 (별도 스펙에서 다룸)
- 알림 히스토리 화면 (필요해지면 추후 추가)
